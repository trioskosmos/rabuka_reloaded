**Created:** 2026-03-16  
**Location:** `frontend/web_ui/`

---

## Quick Start

### Development Mode (with live reload)
```bash
cd frontend/web_ui
npm install              # First time only
npm run dev             # Runs on http://localhost:3000
```

### Build for Production
```bash
cd frontend/web_ui
npm run build           # Creates dist/ folder
npm run preview         # Test the built version locally
```

### After Building: Sync to Backend
```bash
cd <project-root>
python tools/sync_launcher_assets.py
```

---

## What Each Command Does

### `npm run dev`
- **What it does:** Starts dev server with live reload
- **Output:** Runs on port 3000
- **Files served:** Raw from `web_ui/` (HTML/CSS/JS directly, NOT built)
- **CSS changes:** Reflect immediately (HMR—Hot Module Replacement)
- **When to use:** During development

### `npm run build`
- **What it does:** Bundles all assets into `dist/` folder
- **Output:** Optimized, minified code
- **Files included:**
  - All HTML pages (index.html, deck_builder.html, etc.)
  - All CSS bundled and minified (style.css imports are combined)
  - All JS bundled and minified
  - Images referenced in CSS/HTML
  - **NOT included:** node_modules, vite.config.js, source maps (by default)
- **When to use:** Before testing or deploying
- **Time:** ~5-15 seconds

### `npm run preview`
- **What it does:** Serves the built `dist/` folder locally
- **Useful for:** Testing production build before deploy
- **Port:** 4173 (default)

## Vite Configuration Breakdown

### `vite.config.js`

```javascript
root: './',              // Project root is current directory
base: './',             // Assets served from root (not /app/static/)
outDir: 'dist',         // Output goes to dist/
```

### Entry Points
The build creates separate bundles for each page:
- `index.html` → Game board
- `deck_builder.html` → Deck building
- `deck_converter.html` → Format converter
- `deck_viewer.html` → Deck viewer
- `interactive_deck_viewer.html` → Interactive viewer

Each is accessible at its own URL after build.

## Asset Pipeline

### Development Flow
```
Source Files (web_ui/)
    ↓ (Dev server watches for changes)
Browser loads directly (NOT bundled)
    ↓ (CSS hot reload on save)
Changes appear immediately
```

### Production Flow
```
Source Files (web_ui/)
    ↓ (npm run build)
Bundled & Minified (dist/)
    ↓ (python sync_launcher_assets.py)
Backend Static Assets (launcher/static_content/)
    ↓ (Backend serves to frontend)
Browser loads from backend
```

## CSS Loading in Vite

### index.html
```html
<link rel="stylesheet" href="css/style.css">
```

### In style.css
```css
@import './base.css';
@import './layout.css';
/* ... etc ... */
```

### What Happens During Build
1. Vite finds `style.css` import in `index.html`
2. Resolves all `@import` statements
3. Combines all CSS files into single optimized bundle
4. Minifies (removes whitespace, shorthand)
5. Outputs to `dist/css/` or inline depending on size

**Result:** Single HTTP request for all CSS (no waterfall)

## Common Issues & Solutions

### Issue 1: CSS changes don't appear after `npm run build`
**Solution:**
```bash
cd frontend/web_ui
rm -r dist/              # Clean the old build
npm run build            # Rebuild from scratch
```

### Issue 2: CSS changes appear in dev but not in production
**Reason:** Forgot to rebuild  
**Solution:**
```bash
npm run build
python tools/sync_launcher_assets.py
```

### Issue 3: "Module not found: vite"
**Solution:**
```bash
npm install
```

### Issue 4: Dev server won't start on port 3000
**Reason:** Port already in use  
**Solution:** Change port in `vite.config.js`:
```javascript
server: {
    port: 3001,  // Change this
    open: true,
},
```

### Issue 5: Images don't show up in the built version
**Reason:** Image path is relative to wrong directory  
**Solution:** Ensure images are in `frontend/img/` and referenced correctly:
```html
<!-- ✅ Correct -->
<img src="img/card_art.webp" />

<!-- ❌ Wrong -->
<img src="../../frontend/img/card_art.webp" />
```

## Vite vs. Raw Source Strategy

### When sync_launcher_assets.py Chooses Vite Build
```python
VITE_DIST = os.path.join(FE_BASE, "dist")
if os.path.exists(VITE_DIST):
    FE_SRC = VITE_DIST
    print(f"Using Vite build output: {FE_SRC}")
else:
    FE_SRC = FE_BASE
    print(f"Using raw source (Vite build not found): {FE_SRC}")
```

### Implication
- **If `dist/` exists:** Sync uses it (production-ready)
- **If `dist/` missing:** Sync uses raw files (development fallback)

## Environment Variables (Advanced)

To customize Vite behavior, create `.env` file in `frontend/web_ui/`:

```
VITE_DEBUG=1          # Enable debug output
VITE_APP_VERSION=1.0  # Custom version
```

Then reference in code:
```javascript
const version = import.meta.env.VITE_APP_VERSION;
```
